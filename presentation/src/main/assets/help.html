Manual - English

Initial Screen

When you launch CIFS Document Provider for the first time, you will see this window. Depending on the version of Android your device is running ( Android 13 and greater ), you will be asked to grant permission to display notifications.

Please allow the notifications to be displayed, this is crucial for the Foreground Service to work properly if enabled in settings, more on that later.

But if you dismiss the request and you would like to enable it later, you can go to Settings→App and from there, turn on the permission to allow the notification.

From the initial screen, you can see three important buttons.

➡️Add new connection

(folder icon at the bottom)

➡️Settings

(gear icon on the top)

➡️Share

(branch icon on the top)

When tapping the Share icon, CIFS Document Provider will open Android’s Files app. Then you can select CIFS Document Provider from the list of available providers, choose the file you want to share. You can share the file with other installed applications or one of your contacts. It is the normal share feature available in Android.

New Connection Screen

Add new connection

If you tapped on the Add new connection icon, you will see the following screen:

From here you can either choose the Host you are trying to connect to from the list, or if it is not detected automatically, you can tap on “Set manually”. You can also “Refresh” the list of found hosts or “Sort” them too from the top icons. Once you select a host or opt to set it manually then you will see the following screen:

ID : This is a UUID/GUID (Global / Universally Unique Identifier), it is generated by the CIFS Document Provider application. When registering for the first time (creating a new connection), a randomly generated UUID is entered as the initial value, which can be changed to any string. It cannot be changed after registration, as it becomes read-only. Also, the connection information ID cannot be duplicated with other connection information IDs. So make sure if you manually changed it, that it is not used again in other connections.

Name: You can name this “connection” whatever you like. This is just to help you differentiate between multiple connections from the main screen.

Storage: In the drop-down box, you can choose from multiple libraries and connections types. Currently CIFS Document Provider supports two different types of connections CIFS / SMB and FTP / FTPS.

For CIFS / SMB connections, there are two different libraries you can choose. They should have the same functionality. But one might be more suited to your needs than the other or might be faster. You can choose between SMBJ and jCIFS-NG libraries. Note that SMBJ does not support the old and deprecated SMB v1, only jCIFS-NG supports that.

For FTP connections, there is currently one library supported. Apache Commons, but here you can choose either FTP insecure connection or FTPS secure connection with TLS encryption.

Note that if you choose FTP / FTPS connection type, the connection options under settings subsection underneath will change automatically.

Default options are for CIFS / SMB connection type since it is selected by default. For FTP / FTPS, more on that later.

➡️        Under Settings subsection:

Domain: Here you can enter the Domain name if your CIFS / SMB share is part of a workgroup or the user is part of AD (Active Directory). This is for advanced use cases and meant for business environments or corporations. Normal home users can leave this empty. It is optional for home users.

Host: Here you must put the IP address or Hostname that you are trying to connect to.

Port: You can leave this empty by default, unless your network has different configured ports for this connection type. CIFS / SMB usually use ports 139 and 445. While FTP / FTPS usually use ports 20 and 21. If you leave this field empty, it will use one of these default values depending on the connection type.

DFS: If you ticked this check box, you will enable the Distributed File System feature.

Normal home users don’t need to enable this since it is useful for corporate environments. Such as having many computers with a shared CIFS / SMB connection, each computer can have part of the data.

One file can be saved partially on many computers and under DFS (when configured by the Administrator, all computers can access the shared data). The sum of all data shared by all  computers will result in a complete dataset. Just to make it clearer, (it is not a 100% correct analogy) think of this like torrents file sharing, although it is not technically the same.

User: Here you need to put the Username for this connection. If the Anonymous option is enabled, then you don’t need to put the username or the password for the connection.

Note that for CIFS / SMB, the Username is case insensitive, that means username “Poweruser” is the same as “powerUser” and the same as “POWERUSER”.

For FTP / FTPS connection types, it is the opposite, the username is case sensitive. So please make sure you type it correctly.

Password: This is the pass key for this connection, it is case sensitive. That is, if the Password is “Orange”, it is not the same as “oRange” nor “ORANGE”. Please make sure you type this correctly.

Anonymous:Enable this only if your network allows public access without a username or password.

Folder option: Here you can specify the starting path from path (root directory of share or `/` if left empty) or nested path of the share. You can tap on the folder and choose the starting point of the connection, so when other applications trying to access your connection Share, they will see this directory and cannot see anything above this directory (parent directories will become inaccessible).

➡️        Options subsection:

Safe Data Transfer: This is valid for CIFS / SMB connections only, either Enabled or Disabled.

➡️Enabled: Slow sequential reading. But stable random access. It uses library functions directly. On some use cases, and the library chosen, it could be faster than when this setting is disabled.

➡️Disabled: Fast sequential reading, But unstable random access. It uses buffers and read-ahead. But it might have some problems.

You can try both and see which one is better for your needs. Some libraries might be better than the others with this option enabled or disabled.

Read Only: As the name suggests, if this is enabled, any write operation will fail. This is useful if you want to disallow modifications to the network share. Note that this has nothing to do with the assigned permissions for the user for network share.

Add Extension when Saving: This is only useful if the third party application is opening a file using CIFS Document Provider, and when closing the file, it does not add file extension when saving the file. In that specific case, CIFS Document Provider will try to deduce the file extension from the file type and add it to the file name.

If you choose FTP / FTPS connection type, you will see the following changes:

Host: Same as CIFS /SMB. You can put an IP address directly here. The only difference, you can also put the Dynamic DNS address of your host (URL).

For example, say you want to access your FTPS server, which is your NAS server at home. For the Host, you put either the Public IP address that is given to you by your ISP (Internet Service Provider) or if you configured a Dynamic DNS which reflects your Public IP address even if the ISP does change it. Then you can use that instead of an IP address, it will be a URL that always points to your router’s Public IP address.

To elaborate, for example, you can get free Dynamic DNS from:

https://freedns.afraid.org/

But this requires some extra work to get it working, like a script that runs on intervals on your server to update the Dynamic DNS if the Public IP changes. Or ideally, if you have for example a Netgear router, you might be able to create a Dynamic DNS within the router for free using No-IP.com service. In that case, you don’t have to have a script that runs on your server to update the dynamic dns for you, your router will do that for you.

Thus, you can have something like:

subdomain.crabdance.com

or

subdomain.mynetgear.com

That points to your Public IP, there are many sites offering free and paid Dynamic DNS service. And you can use the URL instead of an IP address as a Host.

Port: Same as what was explained above Leave it empty to use default port 20 or 21. If your FTP / FTPS server has a different port configured for the connection, use that number.

➡️        Please make sure that in your Firewall and router, you allow the FTP / FTPS ports to be forwarded. Usually 20 or 21 unless your server has different port numbers. This will suffice for local connections (FTP / FTPS Active connection).

But if you want to use FTP / FTPS connection remotely (from WAN). You also need to open at least a range that spans 10 ports or more for Passive FTP / FTPS connection for optimal performance. Any less and you might have connection stability issues.

In that case, you must use a range of “ephemeral” ports. Also known as User Ports or as  registered ports, include ports from 1024 to 49151. For example, in your Router, allow port forwarding for your FTP / FTPS server IP address from 10000-11000 for TCP connection. And name that Passive FTP connection. Make sure you put that range of Passive connection ports in your FTP / FTPS server’s configurations too.

For CIFS / SMB, usually there is no need to do port forwarding unless you are using different port numbers than the default.

➡️        Also, if you want to connect remotely to your FTP / FTPS server. If you have a misconfigured FTP / FTPS server that doesn’t report back the external IP (public IP), it is possible for the connection to “Succeed” but fail to get any directory or files listing. In that case, look for an option in your FTP / FTPS server to add your Public IP address or Dynami DNS URL to be reported back to the client. Usually, this option will be called “Masquerade Address”, for example, in TrueNAS FTP server. Usually not needed, but worth mentioning for successful connection.

User, Password and Anonymous: The same explanation as what was mentioned above, again, please pay attention that the Username is case sensitive here.

Active Mode: FTP / FTPS connections have two modes, Active or Passive.

Usually, but not necessarily “Active” connections are for local connections within the same network (LAN). And “Passive” when you are trying to access the server from a different network (remotely, i.e, WAN). If you have a problem connecting to your server with Active mode disabled, try enabling it. Default is disabled.

Character Set: Default is UTF-8, it is a character encoding (think of it as a letter or symbol on your keyboard if you will, in computer parlance). Sometimes, you might have problems accessing files with non-English names.

This setting needs  to match what your server’s setting is. Usually, UTF-8 is a Unicode character encoding capable of handling file names with non-english characters without any issues.

Active Mode: If you are not able to successfully connect to your FTP / FTPS server, try enabling this setting.

Implicit SSL / TLS Mode: If your FTP / FTPS server requires Implicit SSL / TLS encryption, then enable this. Otherwise, leave it empty for Explicit mode.

Folder Option, Read Only and Add Extension When Saving : Same as explained eariler.

The information at the bottom is to tell you what path (URI) other third party applications will see. Nothing to configure here.

➡️         Top Bar

On the top of the Screen, there is a Trash Can icon, use this to “Delete” an already saved connection.

Next to it, there is an icon of a Folder with a check mark inside of it, and a small question mark above it. This is to “Test” the connection with the provided information. Saving the connection does not test its validity by default. So make sure to test it to make sure you have a working document provider.

Going back to the Initial Screen, from there we can access the Settings menu.

Settings:

Under Settings subsection:

Help: This will display this help page.

Theme: Here you can change the look and feel

of the application. You can choose between Light, Dark and System Default.

Language: You can change the language of CIFS Document Provider. Currently there are five languages: English, Japanese, Arabic, Chinese and Slovak.

Open File Limit: Default 15 files. You can specify how many files are allowed to be opened concurrently (in parallel). Useful if there is a third party application trying to read or enumerate thousands of files in a directory (for example, a music player adding ID3 tags) or emulator trying to enumerate game discs or ROMs..etc!

Maximum is 999 files. Which is close to the Android OS limit of 1024 Files. But be careful, a very high number can cause crashes. Usually It is safe to leave it at 15, not too low, and not too high.

If you change the value of this setting, an application restart is required for this setting to take effect. You can do this easily when tapping the “App” at the bottom from this Settings screen. Then you choose “Force Close”. Simply exiting the App is not enough since it might still be resident in memory.

Show Notifications When Opening Files: Either Enabled or Disabled. Default is disabled.

This is useful to maintain CIFS Document Provider’s process, to make it resilient to being terminated by Android OS under heavy load.

For example, if there is a third party application reading a file that does intensive calculation, say you are playing a game using an emulator, Android OS might try to kill CIFS Document Provider to save resources. If that happens, you might not get more than 10 minutes worth of gaming and the emulator might freeze and give you an error about Disc read errors.

This option, when Enabled, will create a foreground service internally using Work Manager, this will tell Android to leave the CIFS Document Provider process alone and to keep it going. The foreground service is guaranteed to stop when “All” opened files handlers are closed, so this option is safe to enable and won’t consume battery usage. In fact, it makes CIFS Document Provider more reliable.

Use As Local Storage: Either Enabled or Disabled. When enabling this option, CIFS Document Provider will mimic and emulate internal device storage.

Some third party applications only expect internal storage to open files for reading and writing. They can’t use Cloud services (like DropBox or GDrive for example) or anything else.

Since CIFS Document Provider is by nature accessing files in the network. We need to fool these third party applications into thinking that they are dealing with internal storage.

It is safe to enable this option, but it is not guaranteed to work for all applications. You might be surprised that your favorite application is seeing your files.

For example, lots of game system’s emulators expect a local storage to access game files. When this is enabled, most can access them like normal.

If you change the value of this setting, please reboot your device for the setting to take effect.

➡️        NOTE: This will not create a mount point to the internal storage as a folder.

Libraries: this will display all used libraries to create this application.

Contributors: It will open a page on the Github site that will show a list of all the wonderful people who worked on this application. Thanks to all of them. And thanks to wa2c for this wonderful application.

App: When tapping on this, it will open the Android’s application settings specifically for CIFS Document Provider. From here, you can do various things. You can Force Close the application or clear its internal storage and start on a clean slate.

概要

CIFS Documents Providerの利用方法について説明します。

本アプリはSAF(Storage Access Framework)を介して、外部アプリにオンラインストレージへのアクセス機能を提供します。SAFには、ファイル読込み、ファイル書込み、フォルダアクセスの3種類のアクセス方法がありますが、いずれの方法でも利用することができます。

主な機能と操作手順

ストレージの登録

まず最初に、外部アプリに公開するオンラインストレージの接続情報を、本アプリに登録する必要があります。
ステージ接続情報を登録する手順を次に示します。

1.  メイン画面の追加ボタンを押して編集画面を開きます。

2. 編集画面で、接続先のオンラインストレージ接続情報を入力します。接続チェックボタンを押して接続可能であることを確認し、保存ボタンを押します。

3. メイン画面の一覧に、追加した接続情報が表示されている確認します。

外部アプリからの利用

登録したオンラインストレージは、外部アプリからSAFのピッカーを通じて利用することができます。
外部アプリから利用する手順を次に示します。

1.  外部アプリから、SAFピッカーを開きます。(図参照)

2.  ピッカーのメニューから「CIFS Documents Provider」を選択します。

3.  登録したオンラインストレージが一覧に表示されるので、利用する対象を選択します。

4. 選択したストレージから、操作対象のファイル・フォルダを選択します。

➡️ ファイル読込みの場合は、対象のファイルを選択します。

➡️ ファイル書込みの場合は、保存先フォルダでファイル名を入力します。

➡️ フォルダアクセスの場合は、対象のフォルダを選択します。

ファイル共有

オンラインストレージのファイルを、外部アプリに共有することができます。
ファイルを共有する手順を次に示します。

1.  メイン画面の共有ボタンを押し、SAFピッカーを開きます。

2. ストレージから、対象のファイルを選択します。(複数選択可)

3. 共有先アプリの選択画面から、目的のアプリを選択します。

ファイル転送

アプリに共有されたファイルを、ストレージに転送(コピー)することができます。
ファイルを転送する手順を次に示します。

1.  外部アプリよりファイルを共有します(複数可)。

2.  共有先アプリの選択画面から、CIFS Documents Providerを選択します。

3.  SAFピッカーが表示されるので、転送先を選択します。

➡️ 単一ファイルを選択した場合は、ファイル書込みピッカーになります。

➡️ 複数ファイルを選択した場合は、フォルダ選択ピッカーになります。

4.  選択後、ファイル転送画面が開き、ファイル転送を開始します。

FAQ

利用アプリからSAFのピッカーに表示されません。

ご利用のアプリが、利用可能なDocumentsProvidedrをローカルストレージに限定しています。設定画面で「ローカルストレージとして使う」を有効にすると、アクセスできるようになります。ただし、ご利用のアプリがローカルストレージ上の動作を想定しているため、正常に動作しない可能性があります。

長時間ファイルを利用していると、突然ファイルにアクセス出来なることがあります。

AndroidのシステムによってCIFS Documents Providerのアプリが強制終了されると、それに伴いDocumentsProviderも終了し、ファイルにアクセスできなくなります。設定画面で「ファイル利用時に通知を表示」を有効にすると、強制終了を防ぐことができます。

データの保存先に指定するとアプリがクラッシュします。

動的なストリームデータの保存先に指定すると、アプリがクラッシュします。原因を調査中です。
